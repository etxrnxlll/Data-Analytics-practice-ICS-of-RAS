# 📊 Power BI — Техническая реализация

## Структура файлов

| Файл | Назначение |
|------|------------|
| `dashboard.pbit` | Шаблон отчёта (модель + меры, без данных) |
| `dashboard-demo.pbix` | Демо-версия с синтетическими данными |
| `dax/*.dax` | Библиотека DAX-мер по категориям |
| `power-query/*.m` | ETL-скрипты Power Query |
| `theme/*.json` | Корпоративная цветовая тема |
| `screenshots/` | Визуальная документация отчёта |

## Как открыть

1. Установите [Power BI Desktop](https://powerbi.microsoft.com/desktop/)
2. Откройте `dashboard.pbit` для изучения архитектуры
3. Либо откройте `dashboard-demo.pbix` для интерактивной демонстрации

## Архитектурные решения

### Почему не звёздообразная схема?
Исходные данные поставлялись отдельными файлами за каждый год.
Объединение в единую фактовую таблицу усложнило бы отладку.
Модель на основе изолированных таблиц обеспечивает:
- Прозрачность расчётов
- Независимую модификацию годовых структур
- Отсутствие циклических зависимостей

### Ключевой методологический выбор
**Агрегированные vs средние расчёты:**
- Среднее по организациям искажает результат при разнородных масштабах
- Агрегированный подход учитывает вес каждой организации

## Библиотека DAX

Подробные формулы см. в папке `dax/`:
- 📈 `profitability.dax` — рентабельность и YoY
- 💼 `labor-productivity.dax` — производительность труда
- 💰 `tax-burden.dax` — налоговая нагрузка
- 📉 `loss-ratio.dax` — доля убыточных организаций
- 🔄 `dynamic-year-switch.dax` — переключение периодов
